# Transcript Workflow (Always-On)

Use this workflow at the end of every working session.

1. Open `V2_EXECUTION_TRANSCRIPT.md`.
2. Append a new section:
   - `## Session YYYY-MM-DD`
   - objective, completed items, changed files, tests/checks, blockers, next action
3. Record exact current phase and checkpoint status from `V2_DEPLOYMENT_PLAN.md`.
4. Record the first command/task to run next session.
5. Commit transcript updates with the related work.

Minimum required fields per session:

- Objective
- What was completed
- Files changed
- Current phase + checkpoint
- Next exact action

Rule: Never close a session without updating the transcript.
